Objective

A natural-language autoencoder (NLA) reads a hidden state from a host language model and emits a short natural-language description of its contents. The question this page measures is whether that description recovers the specific fact carried in the hidden state, or instead falls back on whatever the surrounding text and the model's prior make the most likely default — a substituted, confabulated detail in place of the encoded one.

The two outcomes are dissociated by construction. Each item introduces a fact early in the input and later refers back to it with a token held fixed across the two versions. Because that later token and its immediate neighbours are identical regardless of which fact was stated, any version-specific content in its description has to come from the hidden state at that position rather than from the local tokens around it. A description that recovers this version's fact at that token shows the fact is present and decodable in the hidden state there; one that recovers the other version's fact, or neither, shows the read settling on a default instead of the encoded fact.

This is a claim about what the recall-token hidden state contains and whether the decoder surfaces it. It is deliberately not a claim that the decoder reaches information unavailable from the input as a whole: the fact was stated earlier in the same input, so a model that reconstructs it from the broader context would also score as recovering it. What the held-fixed recall token rules out is reliance on the local neighbourhood of the read position, nothing wider.

Scope. The quantities reported on this page derive solely from live reads of the two hosted models on the recall items, which are enumerated in the Callback reads tab. The authored families in the first tab constitute a separate evaluation with a distinct methodology and contribute to none of the figures reported here.

Experimental design

Each item is constructed in two versions that differ in a single fact. For example, one version specifies the hobby as oil painting and the other as sculpture, with all remaining content held constant. This counterfactual pairing isolates the fact as the sole variable across versions.

Each item is instantiated in one of two formats. In the single-message format, the fact and the later reference occur within a single user message. In the conversational format, the user states the fact, the model generates a response, and the user then refers back to the fact; this format additionally exposes read positions within the model's generated response.

Read positions

Within each version, the hidden state is read at several designated tokens. Each class of read is scored separately.

  • Fact token. The token at which the fact is first stated, for example "painting". Because the token itself names the fact, a correct description here is recoverable from the local text alone, so this condition approximates an upper bound rather than a diagnostic read of the hidden state.
  • Recall token. A later token that is identical across the two versions. This is the diagnostic condition: its local context is constant, so a correct description implicates information carried in the hidden state rather than information recoverable from the surrounding tokens.
  • Response token. A position within the model's generated response, available only in the conversational format.

The count reported beneath each result is the number of reads of that class that were sufficiently determinate to score. The fact and recall tokens are present in every item and therefore have the largest counts; response tokens occur only in the conversational items and are correspondingly fewer.

Scoring procedure

This one procedure scores every read in both experiments — the callback recall-token reads (this dashboard's headline test) and the authored counterfactual-family reads (the Families, Family-reads, and Errors & indeterminate tabs). For each read the decoder has already produced a single descriptive sentence; that sentence is scored in isolation, with no access to anything else about the item: not the conversation, not which version it came from, not the reconstruction cosine.

  1. The sentence is presented to three independent language-model graders — a Claude judge panel run as a grading workflow, in batches of twelve reads, three judges per batch. The two options are not invented for grading: they are the item's counterfactual pair — the single fact that was varied to build the two versions, and nothing else. On this dashboard option A is always the pair's first value and option B its second (for the read below, A = “a rescue greyhound”, B = “a tabby kitten”), in a fixed order that does not depend on which version the read came from; the correct option is simply whichever value the read's own version stated. Each grader sees only the sentence, those two options, and an id — not the conversation, the version, the reconstruction cosine, or the other judges' answers. (How the A/B order and the id are set differs between the two experiments — see Both experiments, two safeguards below.)
  2. Each grader answers A, B, or neither, judging each read independently. The instruction is to match on meaning rather than exact words, and to answer "neither" — not guess — whenever a description fits both candidates, fits neither, or is too generic to tell.
  3. The three answers are combined by strict majority: a value is taken only when at least two of the three graders choose it, so a one-to-one-to-one split is recorded as neither.
  4. The majority verdict is compared against the originating version. A verdict matching the true fact is scored correct; the opposing fact is scored wrong; a verdict of neither is recorded as indeterminate and excluded from scoring.

Each grader is given these instructions verbatim: “You are a careful, literal grader. … Match on MEANING, not exact words … Answer ‘A’ if it clearly indicates option A, ‘B’ if option B. Answer ‘neither’ if the description fits both equally, fits neither, or is too generic/abstract to tell. Do NOT guess when genuinely ambiguous. Judge each item independently.”

grounding = correct / (correct + wrong)
Indeterminate reads are reported alongside each bar and are excluded from the denominator.

What each grader sees

Under those instructions the judge gets a numbered list of reads. One entry looks exactly like this — a real callback read, verbatim as the panel received it: the AV's full description, then the two candidate facts as option A and option B (the panel collapses any double quotes in the description to single quotes, shown here as-is):

one item in the batch · callback recall-token read · Gemma · family “pet2”
7. read_id: pet2|v|callback
   description: "UK pet/lifestyle article register established, describing a rescue dog adjusting to a new home, with a positive outcome narrative about a dog named Poppy.

The opening sentence 'he's quite a character and the animal' signals a real-life anecdote about a dog with anxiety/behavioural challenges, likely from a rescue centre.

Final token 'the animal' begins a noun phrase describing a named animal or staff role — 'the animal' strongly expects a noun like 'centre/shelter' or 'centre staff' or 'shelter staff say' or 'rescue told us' or 'carer said' or 'centre/him was amazing' or 'trainer was in his.'"
   option A: "a rescue greyhound"
   option B: "a tabby kitten"
All three judges answered A — the description is plainly about a dog, not a cat, even though it never says “greyhound”. Strict majority → A, and option A (“a rescue greyhound”) is the value the activation actually carried, so this read is scored correct / grounded. A majority of B would be wrong; a 1-1-1 split, or a majority of “neither”, would be indeterminate and dropped from the denominator.

Both experiments, two safeguards

The panel, the verbatim instructions, the strict-majority rule and the grounding formula are identical for the callback reads and the authored-family reads. They differ only in two anti-leak safeguards — both of which the authored-family grading had from the start, and the callback grading gained in the re-grade audit below:

  • Opaque read id. The callback item above shows read_id: pet2|v|callback, whose |v| encodes the version. The authored-family reads instead carry an opaque hash (e.g. 798f044e02d7), so the id leaks nothing.
  • Randomized A/B order. On the callback reads option A is always the first value, B the second. On the authored-family reads the two values are shuffled per read by a hash of the id, so position is uncorrelated with the answer. The same family graded under both versions shows it directly:
randomized order · authored family “wealth” · true value = poor in both versions
wealth_a    option A: "wealthy"    option B: "poor"       → judges: B → “poor”  → correct
wealth_b    option A: "poor"       option B: "wealthy"    → judges: A → “poor”  → correct

Because a family's v and v′ reads are shuffled independently, any A/B position bias cancels in the v-vs-v′ contrast instead of adding to it. One more authored-only detail: Qwen (the local read arm) produces ten sampled descriptions per read, each graded by the panel; the ten are reduced to one read verdict from the committed draws — grounded if at least half point to the true value, wrong otherwise, indeterminate if none commit. Llama and Gemma are hosted single-read, so their read verdict is just the panel's majority on that one description.

Blind re-grade audit

Every contested recall-token read — each read the panel scored wrong or indeterminate, 57 in all — was re-scored by a second blind three-judge panel, now with opaque identifiers and a per-read randomized candidate order, and each stored description was reverified against the live Neuronpedia cache (all 57 reproduced exactly, and the identity mse = 2(1 − cos) held in every case). The two validity-critical directions were clean: no read moved between correct and wrong in either direction, so the re-grade neither exonerated a wrong read nor manufactured one. It revised eight contested reads; those revisions are folded into the figures on this page, and the revised reads are flagged in the Errors & indeterminate tab.

The second panel was somewhat stricter, abstaining on 8 of 24 grounded controls it also re-graded, so the absolute grounding rate should be read as panel-dependent at the level of a few percent. Only the contested reads received complete re-grading; the grounded totals remain the first panel's, and a fully blind re-grade of all reads was not performed. The cross-model comparison and the failure analysis below are unaffected by this.

Reconstruction cosine and its limitations

In addition to grader-based grounding, an automatic reconstruction cosine is reported for each read. A hidden state is reconstructed from the description and compared against the original by cosine similarity; the value lies on a scale with maximum 1, and higher values are intended to indicate more complete descriptions.

Provenance. Every contested read in the audit — all 57 scored wrong or indeterminate, 27 on Gemma and 30 on Llama — was reverified read-for-read against the live Neuronpedia cache: the stored cosine, mse, token, and description reproduced exactly in every case, and the identity mse = 2(1 − cos) held for all of them. The stored quantities are faithful to the source.

Incomparability across models. The raw cosine is not comparable between the two models. Gemma's residual stream is strongly anisotropic: almost all of the magnitude of any hidden state lies along a single direction shared by nearly all tokens. Under such anisotropy, arbitrary pairs of hidden states exhibit high cosine similarity, so the metric is elevated even for an unrelated description. Empirically, a deliberately mismatched description attains a cosine of approximately 0.94 on Gemma but only approximately 0.17 on Llama. Gemma's cosine therefore occupies a narrow band near the ceiling before any genuine readout occurs, leaving little dynamic range to distinguish a faithful description from an unfaithful one.

To correct for this, the tables below report, for each read site, the raw cosine, the floor (the cosine attained by a mismatched description on that model), and an adjusted value defined as (cos − floor) / (1 − floor), the fraction of the available range above the floor that is recovered. After this correction, Llama recovers a larger fraction of the hidden state than Gemma at every read site, reversing the ordering implied by the raw cosine.

Why the adjustment is coarse, especially for Gemma. The correction divides by 1 − floor. For Llama that denominator is about 0.84, a benign factor of roughly 1.2×; for Gemma it is about 0.06, so the correction magnifies the slender gap above the floor about 17×. A change of 0.01 in either the raw cosine or the estimated floor therefore moves Gemma's adjusted value by about 0.17. Gemma's adjusted numbers consequently carry wide uncertainty, and per-read adjusted recovery (shown beside each read in the Errors & indeterminate tab) should be read as indicative only. For this reason the correction is applied to per-site means, where read-to-read noise averages out, rather than trusted read by read.

The floor values are drawn from a separate activation-space study of the same models and layers, so the adjusted values are an approximate correction, not an exact measurement. The qualitative reversal — Llama above Gemma at every site — is the robust conclusion (Gemma's best-adjusted site still sits below Llama's worst), whereas the exact magnitudes, Gemma's in particular, depend on the floor estimate. The grounding rate above remains the reliable cross-model comparison: it is set by the graders and is independent of this geometry.

Where it fails, and why

When a read at the recall word goes wrong, it goes wrong in one of two ways, and the two are worth telling apart.

Unclear, and set aside. Most of the misses are cases where the description names the general situation but never picks the specific thing. It will say "someone talking about their recovery" or "a person looking after a houseplant" without ever choosing knee or shoulder, or orchid or cactus. These tend to happen when the thing being referred to is abstract or hard to picture, such as a plant, a gift, a musical instrument, a piece of furniture, or a place. The description is not wrong, it simply does not commit, so it is left out of the percentage.

Wrong, which is the more revealing case. A smaller number of reads commit to the wrong answer, and when they do, they almost always pick the more typical member of the category. When the hidden state does not carry strong evidence, the model falls back on what is usually true. Often it writes that typical answer out in full: for an example about tea it says "a cup of coffee", for an example about fog it says "the rain has ruined my hike", and for an example about wine it says "a small glass of whiskey". Sometimes the typical answer it reaches for is a third thing that is neither of the two choices, such as reading a cello as a violin, which then counts as unclear rather than wrong.

The same items trip up both models. Fog read as rain, and wine read as whiskey, happen for Gemma and Llama alike. That suggests a shared assumption about what these categories usually are, rather than a quirk of one model. The description leans on that assumption exactly when the hidden state does not pin the answer down.

Errors & indeterminate reads

Objective

This page reports a direct test of whether the verbalizer reads the content of a hidden state. The test proceeds by reading, not by adding any vector: for an authored sentence, the verbalizer reads the hidden state the sentence produces at a chosen anchor position and describes it. The value is carried into that hidden state only by the wording of the sentence. If the description names this version's value, the read recovered the encoded content. If it does not, the description is governed by something other than the read activation, most often the model's prior expectation given the surrounding text.

The items are the authored families enumerated in the first tab. Each family is a counterfactual pair: two values for a single attribute, for example a professional domain that is either agricultural or forestry, sharing one read anchor whose neighboring tokens are held constant. The first version is read at the anchor, then the second version is read at the same anchor. Because the anchor and its surrounding context do not change between the two versions, any difference between the two descriptions must originate in the read activation rather than in the surrounding text.

Recovery is the difference between how often the verbalizer indicates a value when that version is the one read and how often it indicates the same value when the alternative version is read, measured net of a text-only floor.

recovery = P(verbalizer indicates v | version v read) − P(verbalizer indicates v | version v′ read)
Recovery lies between −1 and +1. A value near +1 means the description reliably follows the read activation, which is a grounded read. A value near 0 means which version is read makes no difference and the description reflects the prior. A negative value means the description systematically indicates the opposite of the version's value.

Scope. The figures on this page come from the authored-family read experiment on the qwen7b autoencoder, run locally. This is a separate evaluation from the cross-model dashboard, which uses the two hosted models and a reveal-then-recall design on live reads. The two experiments examine the same question, whether a read reflects the activation or the prior, but they use different models, different items, and different metrics, and their numbers are not interchangeable.

Model and grading

This experiment grades differently from the cross-model dashboard, and the two are not interchangeable. Each verbalizer description is scored by an independent grader model of a different lineage from the autoencoder's base model, asked a constrained multiple-choice question. The grader sees only the description and three options — the value v, the value v′, and neither / unsure as an explicit third choice — and must reply with a single letter.

To cancel position bias rather than assume it away, the question is asked k times with the option order permuted each run (a fixed per-item schedule); each answer is de-permuted back to its role and the majority over the k runs is taken. A tie, an unparseable answer, or a grader that declines all map to neither, never to a value. The estimate then climbs one level at a time: the k-run majority gives one indication per draw; a majority over the draws gives one indication per (condition, wording) cell; the rate over wordings gives P(indicate v | condition); and the per-family signal is P(indicate v | version v read) − P(indicate v | version v′ read), taken net of the text-only floor. The family is the unit of evidence; draws and wordings are nested and never inflate the count. The exact grader model, number of runs, draws, bootstrap resamples, and permutations for this run are listed below.

The grader prompt is, verbatim: “Read the following description, then answer the question. Description: {description} Question: Based ONLY on this description, what does it indicate about the writer's likely attribute value? (a) {one value} (b) {another value} (c) {neither / unsure} Answer with the single letter of the best option.” The three options carry the two values and “neither / unsure” in a per-run-permuted order.

Certified-grounded requires all of: the bootstrap CI excludes zero (lower bound > 0), the point recovery is at least 0.15, the family permutation test is significant after family-wise correction (Holm at a 0.05 error rate for confirmatory attributes, Benjamini-Hochberg at a 0.10 false-discovery rate otherwise), the effect is symmetric across both read directions (v and v′), inter-rater agreement (Cohen's κ) is at least 0.6, and the premise gate is clean. A CI that includes zero is prior-driven; anything in between is activation-specific but unverified. These boundaries are fixed in code, not tuned per run.

Premise gate

A family is admitted to the read-side test only if it first passes a premise gate. The gate enforces three conditions, so that a recovered value can be attributed to the read activation rather than to an artifact.

  • Presence. The base model must already encode the attribute at the read anchor — the activations there must separate the two values at an accuracy of at least 0.67 (chance is 0.5). If the value cannot be read from the activation at all there is nothing for the read to recover, and the family is dropped.
  • Text-only floor. Given only the shared suffix text and no activation read, the grader must not be able to guess the value. A floor near zero confirms that the surrounding text does not give the answer away on its own.
  • Wiring. A positive control confirms that the read path actually carries the activation's content to the verbalizer, so that a null result reflects the model and not a broken pipeline.

Results by attribute

Each row reports the recovery for one attribute, its bootstrap confidence interval, the raw permutation p-value computed with the family as the unit, the family-wise adjusted p-value that the verdict actually uses, and the number of families. What the verdicts mean

Certified-grounded. The reading of this attribute genuinely tracks the read activation. The effect is positive, large enough to matter, statistically significant across families after correction, symmetric across the two read directions, reliably graded, and supported by a clean premise gate. This is the strong positive result.

Activation-specific, unverified. There is a positive, activation-specific effect, but it does not clear the full bar. Most often this is because too few families were available for the corrected permutation test to reach significance, or the effect is below the minimum size. This is suggestive, not established.

Prior-driven. The apparent reading is explained by the model's text prior rather than by the read activation. The confidence interval includes zero, so reading the alternative version did not change what the grader could read.

Certification requires all of the following: a confidence interval excluding zero, a point estimate of at least 0.15, a significant family-wise adjusted permutation test, symmetry across the two read directions, an inter-rater agreement of at least 0.6, and a clean premise gate.

The recovery metric, and what limits it

Recovery is a behavioral measurement. It records whether independent graders could read the version's value from the description, netted across the two read directions. It is not an activation-space cosine. For this reason the anisotropy that complicates the reconstruction cosine on the cross-model dashboard, where a mismatched description can already score near the ceiling, does not apply here. Recovery is bounded between −1 and +1, a value of zero means which version is read made no measurable difference, and the text-only floor and the wiring control anchor the scale so that it cannot be inflated by the geometry of the residual stream.

The binding limitation is statistical power, not geometry. The permutation test uses the family as its unit, so an attribute represented by only a few families cannot reach significance even when its point estimate is large.

An unverified verdict should therefore be read as underpowered, not as disproven. Adding families to these attributes is the direct way to resolve them.

Does reconstruction fidelity predict a misread?

Every read carries a reconstruction fidelity: the cosine (equivalently the MSE) between the activation the verbalizer read and the activation an independent reconstructor re-encodes from the verbalizer's description. The intuitive worry is that the errors and indeterminate reads are just the low-fidelity ones — that a poor round-trip is what produces a wrong or non-committal description, so cosine could be used to screen reads. This tab tests that hypothesis directly, within each model, on every graded read in both experiments.

How this is measured

  • Unit = one read. Callback has one read per item-version; the authored qwen arm draws ten descriptions per read, each with its own cosine and its own grade, so the draw is the unit there. Llama and Gemma are single-read.
  • Within model only. The residual stream is anisotropic and raw cosine is not comparable across models (see the dashboard cosine box), so each test runs inside one model. The statistic is a rank measure, so it is unchanged by the monotonic floor-correction — the result is identical for raw and adjusted cosine.
  • Separation = AUC. The probability that a randomly chosen grounded read has a higher cosine than a randomly chosen not-grounded (wrong or indeterminate) read. 0.5 means fidelity tells you nothing; above 0.5 means grounded reads reconstruct better (the intuitive hypothesis); below 0.5 means grounded reads reconstruct worse.
  • Inference. A 95% confidence interval from a bootstrap that resamples whole families, so the ten draws nested in a read and the two versions of a pair are not double-counted as independent evidence. An interval clear of 0.5 is significant in that direction. A naive Mann-Whitney p (which assumes independence, so it is anti-conservative here) is shown alongside for reference.

MSE is not analysed separately: across all reads the identity mse = 2(1 − cos) holds to within 1e-5, so MSE and cosine are one signal and every cosine result mirrors exactly in MSE.
