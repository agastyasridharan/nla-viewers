Objective

A natural-language autoencoder (NLA) reads a hidden state from a host language model and emits a short natural-language description of its contents. The central validity question is whether such a description reflects genuine readout of the information encoded in the hidden state, or whether it is an instance of input inversion.

Input inversion denotes the failure mode in which the decoder reconstructs the tokens in the local neighborhood of the read position rather than decoding what the hidden state represents. Because a hidden state at a given position is determined largely by the preceding tokens, a description obtained by inverting that local context can appear substantive while conveying nothing beyond what the surrounding text already specifies. Separating genuine readout from input inversion is the purpose of the experiment reported here.

The two cases are dissociated by construction. Each item introduces a fact early in the input and later refers back to it with a token that is held fixed across conditions. The local context of that later token is therefore identical regardless of the fact, so any version-specific content in its description cannot have been produced by inverting the local input and must instead have been carried in the hidden state. A description that recovers the fact at this token is evidence against input inversion; a description that fails to recover it is consistent with input inversion.

Scope. The quantities reported on this page derive solely from live reads of the two hosted models on the recall items, which are enumerated in the Callback reads tab. The authored families in the first tab constitute a separate evaluation with a distinct methodology and contribute to none of the figures reported here.

Experimental design

Each item is constructed in two versions that differ in a single fact. For example, one version specifies the hobby as oil painting and the other as sculpture, with all remaining content held constant. This counterfactual pairing isolates the fact as the sole variable across versions.

Each item is instantiated in one of two formats. In the single-message format, the fact and the later reference occur within a single user message. In the conversational format, the user states the fact, the model generates a response, and the user then refers back to the fact; this format additionally exposes read positions within the model's generated response.

Read positions

Within each version, the hidden state is read at several designated tokens. Each class of read is scored separately.

  • Fact token. The token at which the fact is first stated, for example "painting". Because the token itself names the fact, this condition approximates an upper bound rather than a test of input inversion.
  • Recall token. A later token that is identical across the two versions. This is the diagnostic condition: its local context is constant, so a correct description implicates information carried in the hidden state rather than information recoverable from the surrounding tokens.
  • Response token. A position within the model's generated response, available only in the conversational format.

The count reported beneath each result is the number of reads of that class that were sufficiently determinate to score. The fact and recall tokens are present in every item and therefore have the largest counts; response tokens occur only in the conversational items and are correspondingly fewer.

Scoring procedure

For each read, the decoder has already produced a single descriptive sentence. That sentence is scored in isolation, with no access to any other information about the item.

  1. The sentence is presented to three independent language-model graders. Each grader receives only the sentence and the two candidate facts, designated A and B. The graders do not observe the conversation or the version from which the read originated, and the candidate ordering is held constant across versions, so ordering is uninformative.
  2. Each grader returns A, B, or neither. The majority verdict over the three graders is taken; a three-way disagreement is recorded as neither.
  3. The majority verdict is compared against the originating version. A verdict matching the true fact is scored correct; the opposing fact is scored wrong; a verdict of neither is recorded as indeterminate and excluded from scoring.

grounding = correct / (correct + wrong)
Indeterminate reads are reported alongside each bar and are excluded from the denominator.

Per-model results

The reported figure is the grounding rate at the recall token, defined as the proportion of determinate reads whose description the graders matched to the correct fact. Results for Gemma and Llama are reported separately below.

Grounding by read site

Each bar gives the grounding rate for the corresponding read site. The fact token sits on the answer and therefore approximates an upper bound. The recall token is the diagnostic condition: because it is identical across versions, a correct description indicates that the fact was carried in the hidden state rather than recovered by input inversion.

Reconstruction cosine and its limitations

In addition to grader-based grounding, an automatic reconstruction cosine is reported for each read. A hidden state is reconstructed from the description and compared against the original by cosine similarity; the value lies on a scale with maximum 1, and higher values are intended to indicate more complete descriptions.

Provenance. The recorded values are veridical. Every stored cosine was reverified against the live Neuronpedia service: across a sample of 200 reads, 100 per model, all values reproduced exactly, and the identity mse = 2(1 − cos) holds throughout. The stored quantities are therefore faithful to the source.

Incomparability across models. The raw cosine is not comparable between the two models. Gemma's residual stream is strongly anisotropic: almost all of the magnitude of any hidden state lies along a single direction shared by nearly all tokens. Under such anisotropy, arbitrary pairs of hidden states exhibit high cosine similarity, so the metric is elevated even for an unrelated description. Empirically, a deliberately mismatched description attains a cosine of approximately 0.94 on Gemma but only approximately 0.17 on Llama. Gemma's cosine therefore occupies a narrow band near the ceiling before any genuine readout occurs, leaving little dynamic range to distinguish a faithful description from an unfaithful one.

To correct for this, the tables below report, for each read site, the raw cosine, the floor (the cosine attained by a mismatched description on that model), and an adjusted value defined as (cos − floor) / (1 − floor), the fraction of the available range above the floor that is recovered. After this correction, Llama recovers a larger fraction of the hidden state than Gemma at every read site, reversing the ordering implied by the raw cosine.

The floor values are drawn from a separate activation-space study of the same models and layers; the adjusted values should be interpreted as an approximate correction rather than an exact measurement. The grounding rate above provides the reliable cross-model comparison, as it is determined by the graders and is independent of this geometry.

Where it fails, and why

When a read at the recall word goes wrong, it goes wrong in one of two ways, and the two are worth telling apart.

Unclear, and set aside. Most of the misses are cases where the description names the general situation but never picks the specific thing. It will say "someone talking about their recovery" or "a person looking after a houseplant" without ever choosing knee or shoulder, or orchid or cactus. These tend to happen when the thing being referred to is abstract or hard to picture, such as a plant, a gift, a musical instrument, a piece of furniture, or a place. The description is not wrong, it simply does not commit, so it is left out of the percentage.

Wrong, which is the more revealing case. A smaller number of reads commit to the wrong answer, and when they do, they almost always pick the more typical member of the category. When the hidden state does not carry strong evidence, the model falls back on what is usually true. Often it writes that typical answer out in full: for an example about tea it says "a cup of coffee", for an example about fog it says "the rain has ruined my hike", and for an example about wine it says "a small glass of whiskey". Sometimes the typical answer it reaches for is a third thing that is neither of the two choices, such as reading a cello as a violin, which then counts as unclear rather than wrong.

The same items trip up both models. Fog read as rain, and wine read as whiskey, happen for Gemma and Llama alike. That suggests a shared assumption about what these categories usually are, rather than a quirk of one model. The description leans on that assumption exactly when the hidden state does not pin the answer down.

Objective

This page reports a direct test of whether the verbalizer reads the content of a hidden state. The test proceeds by injection. For an authored sentence, the hidden state at a chosen position is replaced with an activation that encodes a specific value, and the verbalizer is asked to describe that position. If the description names the injected value, the read recovered the injected content. If it does not, the description is governed by something other than the injected activation, most often the model's prior expectation given the surrounding text.

The items are the authored families enumerated in the first tab. Each family is a counterfactual pair: two values for a single attribute, for example a professional domain that is either agricultural or forestry, sharing one read anchor whose neighboring tokens are held constant. The activation encoding the first value is injected and the anchor is read, then the activation encoding the second value is injected and the same anchor is read again. Because the anchor and its context do not change between the two injections, any difference between the two descriptions must originate in the injected activation rather than in the surrounding text.

Recovery is the difference between how often the verbalizer indicates a value when that value is injected and how often it indicates the same value when the alternative is injected, measured net of a no-injection floor.

recovery = P(verbalizer indicates v | v injected) − P(verbalizer indicates v | v′ injected)
Recovery lies between −1 and +1. A value near +1 means the description reliably follows the injected activation, which is a grounded read. A value near 0 means the injection makes no difference and the description reflects the prior. A negative value means the description systematically indicates the opposite of the injected value.

Scope. The figures on this page come from the authored-family injection experiment on the qwen7b autoencoder, run locally. This is a separate evaluation from the cross-model dashboard, which uses the two hosted models and a reveal-then-recall design on live reads. The two experiments examine the same question, whether a read reflects the activation or the prior, but they use different models, different items, and different metrics, and their numbers are not interchangeable.

Model and grading

Premise gate

A family is admitted to the read-side test only if it first passes a premise gate. The gate enforces three conditions, so that a recovered value can be attributed to the injection rather than to an artifact.

  • Presence. The base model must already encode the attribute at the read anchor. If the value cannot be read from the unmodified activation, there is nothing for the injection to recover, and the family is dropped.
  • No-injection floor. With no injection, the grader must not be able to guess the value from the description. A floor near zero confirms that the surrounding text does not give the answer away on its own.
  • Wiring. A positive control confirms that the injection path actually alters the activation, so that a null result reflects the model and not a broken pipeline.

Results by attribute

Each row reports the recovery for one attribute, its bootstrap confidence interval, the raw permutation p-value computed with the family as the unit, the family-wise adjusted p-value that the verdict actually uses, and the number of families. Rows are ordered by raw permutation p-value, strongest evidence first.

What the verdicts mean

Certified-grounded. The reading of this attribute genuinely tracks the injected activation. The effect is positive, large enough to matter, statistically significant across families after correction, symmetric across the two injection directions, reliably graded, and supported by a clean premise gate. This is the strong positive result.

Activation-specific, unverified. There is a positive, activation-specific effect, but it does not clear the full bar. Most often this is because too few families were available for the corrected permutation test to reach significance, or the effect is below the minimum size. This is suggestive, not established.

Prior-driven. The apparent reading is explained by the model's text prior rather than by the injection. The confidence interval includes zero, so injecting the alternative value did not change what the grader could read.

Certification requires all of the following: a confidence interval excluding zero, a point estimate of at least 0.15, a significant family-wise adjusted permutation test, symmetry across the two injection directions, an inter-rater agreement of at least 0.6, and a clean premise gate.

The recovery metric, and what limits it

Recovery is a behavioral measurement. It records whether independent graders could read the injected value from the description, netted across the two injection directions. It is not an activation-space cosine. For this reason the anisotropy that complicates the reconstruction cosine on the cross-model dashboard, where a mismatched description can already score near the ceiling, does not apply here. Recovery is bounded between −1 and +1, a value of zero means the injection made no measurable difference, and the no-injection floor and the wiring control anchor the scale so that it cannot be inflated by the geometry of the residual stream.

The binding limitation is statistical power, not geometry. The permutation test uses the family as its unit, so an attribute represented by only a few families cannot reach significance even when its point estimate is large.

An unverified verdict should therefore be read as underpowered, not as disproven. Adding families to these attributes is the direct way to resolve them.
