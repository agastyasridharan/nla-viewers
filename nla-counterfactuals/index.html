How this works

When the NLA reads a hidden activation and writes a sentence describing it, we want to know one thing: is it really reading the activation, or just paraphrasing the words next to it? The reveal-early, call-back-later setup is built to tell those apart.

What this scores: only the callback experiment (the “Callback reads” tab data), across the two hosted models (gemma, llama). It does not include the authored families from the first tab. Those are a separate apparatus with a different evaluation method (a local base-model presence gate plus a multiple-choice grader), and none of that feeds these numbers.

  • In scope: the live hosted-NLA reads from the callback items (cat vs dog, Japan vs Italy, and so on).
  • Out of scope: the 28 authored families in tab 1.

Each item runs in two shapes. In mid_single the callback word sits in the middle of a single message. In multiturn it is spread across a real chat: the user reveals the fact, the model writes a reply (generated live), then the user refers back, which also lets us read the model’s own reply.

Which reads are scored

Every callback item is run in two versions (value v and value v′), and each version has a few read sites. All of them are scored and shown separately:

  • condition: the activation at the value word itself (where the fact first appears).
  • callback: the activation at the shared later word that is identical in both versions. This is the real test.
  • assistant_ref / assistant_end: points inside the model’s own generated reply (multiturn items only).

The “n scored” under each bar is how many reads of that type were clear enough to score. For example gemma condition is 66 (33 items × 2 versions); callback is similar; assistant sites only exist for the multiturn items, so their n is smaller.

How the judging works, step by step

  1. For each read, the NLA already produced an AV description: one sentence describing the activation at that token.
  2. We hand that description to three independent LLM judges. Each judge sees only the description plus the two candidate values, labeled option A (= value v) and option B (= value v′). They do not see the transcript, and they do not see which version the read actually came from. The A/B labeling is the same for both versions, so position gives nothing away.
  3. Each judge answers A, B, or neither. We map A → v, B → v′, and take the majority of the three. A three-way split counts as neither.
  4. We compare that consensus to the true version the read came from:
    • consensus matches the truth → grounded (correct)
    • consensus is the other value → wrong
    • consensus is neither → unclear, set aside

grounding shown on each bar = grounded / (grounded + wrong)
unclear reads are reported next to the bar and left out of that percentage.

Reconstruction fidelity (cosine)

Cosine is separate from the judges. Take the description, rebuild an activation from it, and measure cosine similarity to the original. Higher means the description captured more of the activation.

Result at a glance

Grounding by read site

Each bar is the share of clear reads the judges grounded to the right value (longer is better). The condition site sits on the value word itself, so near 100% there is close to an upper bound. The callback site is the real test: the read token is the same in both versions, so getting it right means the early fact was carried forward.

Where it fails, and why

Across both models, callback misses come in two kinds, and they are not the same thing.

Unclear (set aside). Most misses are the description naming the scene but not the thing. It supports “someone talking about their recovery” or “a person’s relationship with a houseplant”, but not knee versus shoulder, or orchid versus cactus. These cluster on referents that are abstract or hard to picture: a plant, a gift, an instrument, a piece of furniture, a place. The description is not wrong, it just does not commit, so it is left out of the percentage.

Wrong (the informative one). A smaller set commits to the wrong value, and it is almost always the more stereotypical member of the category. When the activation evidence is weak, the prior wins. Often the description writes the prototype out in full: for a tea item it says “a cup of coffee”, for a fog item “the rain has ruined my hike”, for a wine item “a small glass of whiskey”. Sometimes the prototype it reaches for is a third option that is neither choice, like reading a cello as a violin, which then lands as unclear rather than wrong.

Both models, same items. Fog read as rain and wine read as whiskey happen for gemma and llama alike. That points to a shared prior about what these categories usually are, rather than one model’s quirk: the description leans on its prior exactly when the activation does not pin the answer down.
